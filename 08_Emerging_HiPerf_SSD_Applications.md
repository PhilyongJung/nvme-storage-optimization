### 3.2 Physical AI의 데이터 폭발

```
Physical AI가 Storage-Intensive한 이유:

  1. 실세계 센서 데이터 = 초고용량
     ─────────────────────────────
     카메라 8-12개 (4K, 30fps): 4-8 GB/min
     LiDAR 1-5개 (포인트 클라우드): 100-300 MB/min
     Radar, IMU, Force/Torque:     10-50 MB/min
     ──────────────────────────────────────────
     단일 로봇/차량:              ~5-10 GB/min = 300-600 GB/hr

  2. 합성 데이터 = 실데이터의 10~1000배
     ──────────────────────────────────────
     Omniverse Replicator: 1000만 프레임 = 50-200 TB (1회 생성)
     Cosmos WFM: PB급 합성 비디오 생성 가능
     Domain Randomization: 조명/텍스처/각도 변형 → ×100-1000 배

  3. 시뮬레이션 = 반복적 대규모 I/O
     ────────────────────────────────
     Isaac Sim: 4096-16384 병렬 환경, 초당 수천 스텝
     DRIVE Sim: 수십억 마일 시뮬레이션 → 시나리오 데이터 TB~PB
     체크포인트: 대규모 모델 × 빈번한 저장 = 수백 TB

  4. 학습 인프라 = GPU 수천 대 × 데이터 피딩
     ──────────────────────────────────────────
     DGX SuperPOD 256+ GPU: 100+ GB/s 스토리지 처리량 필요
     GR00T/Cosmos 학습: PB급 데이터셋 반복 스캔
```

## 10. Video AI / Spatial Computing

### 10.1 비디오 AI

```
규모:
  - 글로벌 감시 카메라: ~10억 대 (2025), 15-20억 대 (2028)
  - 4K 카메라 1대: 50-100 GB/일
  - 100만 대 도시: 50-500 PB/일
  - Tesla 600만 대 × 카메라 데이터: 1-10 EB/년 (필터 후)

NVMe 필수 이유:
  - 24/7 연속 쓰기: 고내구 NVMe (3-10 DWPD)
  - 동시 읽기+쓰기: AI 추론 중 신규 데이터 수집
  - 프레임 단위 랜덤 접근: AI 분석 시 특정 프레임 탐색
  - Edge 배치: 소형 NVMe in 카메라/NVR
```

### 10.2 Spatial Computing (AR/VR)

```
규모:
  - Volumetric Video: 1분당 10-50 GB (기존 4K의 10-50배)
  - 3D 씬 (NeRF/Gaussian Splatting): 1-10 GB/씬
  - 도시급 디지털 트윈: 100+ TB
  - Apple Vision Pro 콘텐츠: 1-2 GB/분

NVMe 필수 이유:
  - 기존 비디오 대비 10-100배 대역폭 필요
  - 3D 데이터 랜덤 접근 (사용자 시선 방향 따라)
  - VR/AR 지연 = 멀미 → sub-ms I/O 필수
  - Edge CDN: NVMe 캐시 in CDN PoP
```
